# 予約自動化システム まとめ

## 目的
自分のWebページに対して、指定した条件に一致する予約枠を見つけて、自動で予約完了まで進める仕組みを作る。

ソースコード自体は手元にないため、開発者モードで見える範囲の情報を使って自動化する前提。

---

## 前提
- 対象ページはブラウザ上の予約ページ
- 一覧表の中に「日付」「時刻」「段階」「メニュー」「予約ボタン」がある
- ログインが必要な場合がある
- 予約遷移先では、席が埋まっていると予約ボタンが表示されない
- 最終的に「完了」などの確定ボタンを押して終了する
- 定期実行したいので cron を使う

---

## フロー
### 1. ログイン
- まず対象URLへアクセスする
- もしログイン画面が表示されたら、ID とパスワードを入力してログインする
- ログイン済みならそのまま次へ進む

### 2. 一覧ページで対象の予約枠を探す
- 引数で指定した条件を使って一覧表を探索する
- 主に使う条件:
  - date
  - time
  - menu
- 一致する行を見つけたら、その行の「予約する」ボタンを押す

### 3. 遷移先ページで予約可能か確認
- ボタン押下後のページに遷移する
- 席が埋まっている場合は、その先の予約ボタンが表示されない
- その場合は「予約不可」と判断して終了する

### 4. 予約完了処理
- 予約可能なら、最後の「完了」「確定」などのボタンを押す
- 完了メッセージや完了画面を確認できたら終了する

---

## 技術選定
## 結論
今回の用途では **Python + Playwright + cron** が最適。

### 理由
ブラウザの開発者ツールの Console に貼る JavaScript でも一部の操作はできるが、以下の理由で本件には不向き:
- ログイン処理をまたぐ
- ページ遷移をまたぐ
- 定期実行したい
- 失敗時のログやスクリーンショットを残したい
- cron から動かしたい

そのため、外部スクリプトからブラウザを自動操作できる **Playwright** を使うのが最も現実的。

---

## 使う技術
### 1. Python
用途:
- 自動化スクリプト本体
- 引数の受け取り
- 条件判定
- 終了コード管理
- ログ出力

### 2. Playwright
用途:
- ブラウザ起動
- ログイン画面判定
- 入力欄への値入力
- ボタンクリック
- ページ遷移待機
- 表の中から条件一致行の探索
- エラー時スクリーンショット保存

### 3. cron
用途:
- 定期実行
- 指定時刻や一定間隔で予約処理を実行

### 4. flock
用途:
- cron 実行時の二重起動防止

---

## JavaScript と Python の役割分担
### JavaScript
向いている用途:
- 開いているページ上で一時的に試す
- 開発者ツールの Console でDOMを確認する
- selector の調査
- 試しにボタンが押せるか確認する

### Python
向いている用途:
- 本番運用
- ログインから完了まで一連の自動化
- ページ遷移をまたぐ処理
- cron による定期実行
- 安定運用

---

## 引数の考え方
スクリプト実行時に予約条件を引数で渡す。

例:
```bash
python reserve.py --date "2026/04/25" --time "9:00" --menu "高速教習"
```

### 主な引数
- `--date`
- `--time`
- `--menu`

これらを使って一覧ページの対象行を特定する。

---

## 画面上で探す情報
一覧表では、基本的に次の列を見て判定する:
- 日付列
- 時刻列
- メニュー列
- 同じ行にある予約ボタン

たとえば:
- 日付: `2026/04/25`
- 時刻: `9:00-`
- メニュー: `高速教習`

に一致する行の「予約する」を押す。

---

## ログイン判定の考え方
以下のような条件でログイン画面かを判定する:
- password入力欄がある
- ID入力欄らしき text入力欄がある
- ログインボタンがある

ログイン画面なら:
- IDを入力
- パスワードを入力
- ログインボタンを押す

---

## 予約不可判定
予約ボタンを押した先のページで、
- 「予約する」
- 「完了」
- 「確定」
などのボタンが存在しない場合は、席が埋まっている可能性が高い

その場合は予約不可として終了する。

---

## 成功判定
以下のいずれかで成功とみなす:
- 最終ボタン押下後に完了画面に遷移した
- ページ本文に完了文言が表示された
  - 例: `完了`
  - 例: `予約が完了`
  - 例: `受付しました`

---

## cron 実行
Python スクリプトは cron で実行可能。

例:
```bash
* * * * * /usr/bin/python3 /home/you/reserve.py --date "2026/04/25" --time "9:00" --menu "高速教習" >> /home/you/reserve.log 2>&1
```

### 意味
- 毎分実行
- ログをファイルに保存
- 標準エラーも同じログへ出力

---

## 二重実行防止
前回の処理が終わる前に次が走るのを防ぐため、`flock` を併用する。

例:
```bash
* * * * * flock -n /tmp/reserve.lock /usr/bin/python3 /home/you/reserve.py --date "2026/04/25" --time "9:00" --menu "高速教習" >> /home/you/reserve.log 2>&1
```

## 事前に開発者モードで確認したいもの
本実装前に DevTools で次を確認するとよい。

### ログイン画面
- ID入力欄の `id` / `name`
- パスワード入力欄の `id` / `name`
- ログインボタンの selector

### 一覧ページ
- `tr` と `td` の構造
- 各列の順番
- 予約ボタンの selector
- URL がページ遷移で変わるか

### 遷移先ページ
- 予約可能時に出るボタンの文字
- 最終完了ボタンの文字
- 成功時の文言

---

## 実装方針
1. まず DevTools で selector を確認する
2. Python + Playwright でログイン処理を実装する
3. 一覧ページで引数条件に一致する行を探す
4. 遷移先で予約可否を判定する
5. 予約可能なら完了まで進める
6. 成功/失敗を終了コードで返す
7. cron + flock で定期実行する

---

## 失敗時の対策
- ログ出力を残す
- エラー時にスクリーンショットを保存する
- 対象行が見つからない場合は明示的に終了する
- 予約不可時も明示的に終了する

---

## 現時点での最終方針
**採用技術**
- Python
- Playwright
- cron
- flock

**理由**
- ログインを含むページ遷移をまたげる
- 一覧ページから遷移先まで一連の操作を自動化できる
- 定期実行できる
- 運用しやすい
- DevTools で確認した情報をそのまま活用できる

---

## 補足
Console に貼る JavaScript は、調査や仮実験には有効。
ただし本番運用は Python + Playwright にまとめるのがよい。
